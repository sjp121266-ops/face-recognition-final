# 人脸识别期末作业

这是一个 Android Studio + Kotlin 的本地离线人脸识别 App。应用使用 CameraX 拍照，使用 ML Kit 检测人脸，使用 TensorFlow Lite 运行 FaceNet 模型提取 128 维人脸特征向量，并在手机本地保存“姓名 + 人脸特征向量”用于后续识别。本机演示模式不会上传人脸图片或特征；只有用户主动切换到云端 API 模式并填写配置后，才会上传裁剪后的人脸图用于课程对比演示。项目目标是稳定完成期末演示，而不是做商业级门禁或金融核身系统。

## 功能

- 拍照录入人员姓名和人脸特征。
- 拍照识别当前人脸是谁。
- 视频多人识别：点击后进入全屏相机演示，准实时分析预览画面，可同时检测多张人脸，在预览上绘制人脸框和姓名/未知标签，并逐张给出身份判断。
- 视频多人识别在至少两张人脸同时入镜、且出现有效稳定身份标签后，才会写入“视频演示”记录，用于测试摘要验收覆盖。
- 未匹配到已录入人员时显示“未知人员”。
- 使用安全区、稳定区、边界区、拒识区解释识别结果。
- 当第一候选人和第二候选人距离过近时拒绝确认，降低误识别风险。
- 视频多人识别采用跨帧跟踪、短窗口投票和平滑显示，减少单帧误判造成的标签跳动。
- 录入/识别前会检查人脸大小、角度、贴边、亮度和清晰度，减少低质量样本影响识别。
- 统一使用带轻微扩边的人脸裁剪策略，减少额头、下巴或边缘被框裁掉导致的特征波动。
- 处理未检测到人脸、检测到多张人脸、相机权限拒绝等情况。
- 本地保存最近识别记录。
- 显示已录入人数、每人录入次数、当前 L2 阈值和演示准备度。
- 主页面只保留人脸识别演示流程；云端配置、识别记录、报告素材和数据管理收纳在“设置与材料”页。
- 首页会根据当前人脸库和识别记录显示“下一步”提示，引导演示者完成录入、识别、未知人员、多人视频和摘要生成。
- 首页、识别结果和全屏多人识别加入更明确的 UI 层级与动效提示：入场状态突出当前演示进度，录入/识别/未知/拒识状态切换保持可读，全屏多人识别强化扫描感和人脸框反馈。
- 测试摘要会自动分析验收覆盖，提示识别成功、未知人员、边界/拒识、候选人过近和视频多人识别等场景是否已完成。
- 自动检查本地人脸库健康状态，异常特征向量会被清理，避免坏数据影响识别。
- 录入、识别和人脸库健康检查都只接受 128 维有限向量，会拦截或清理维度错误、NaN/Infinity 等异常向量，并在识别说明中展示跳过数量。
- 检查同一人员多组样本的一致性，样本差异过大时提示重新补录。
- 检查不同人员之间的区分度，距离过近时提示存在混淆风险。
- 根据同人最大 L2 和跨人最近 L2 生成阈值校准建议，帮助解释默认阈值是否适合当前人脸库。
- 答辩前建议记录同人最大 L2、跨人最近 L2 和建议阈值；默认阈值保持 10.0，不在没有真机样本分布时盲调。
- 人脸库和识别记录可以分别清空，避免演示时误删数据。
- 测试摘要内置本地模式离线证明：模型、人脸库和识别流程都在本机；云端 API 模式作为可选第二引擎。
- “复制完整报告”会复制截图摘要、技术详情、人脸库明细、完整识别记录、云端 API 演示材料、隐私与风险说明和报告使用建议，方便直接整理到课程报告。
- 支持云端 API 模式：可选择 Face++ 托管 API 或自部署 CompreFace REST 服务，形成“本地离线 + 线上服务”两套识别流程。
- 云端 API 模式会保存最近一次连接测试状态，显示测试时间、提供商和成功/失败原因，并把 Key 错误、额度限制、网络失败等常见问题转成中文提示，方便答辩前确认云端第二模式是否可用。

## 交付状态

当前项目已经可以作为期末作业主体版本交付。本机离线模式是最低交付主线，云端 API 模式是可选对比或加分模块。答辩时建议先完整展示本机录入、识别、未知人员、多人视频和断网识别，再视现场网络情况展示 Face++ 或 CompreFace 云端对比。当前终端环境可以调用 `adb`，但尚未检测到已连接的真机或模拟器；真机安装启动仍需在 Android Studio 或可用设备环境中完成并记录结果。

便于提交和传输的 Debug APK 副本位于项目根目录 `人脸识别演示台-debug.apk`；Gradle 原始输出仍位于 `app/build/outputs/apk/debug/app-debug.apk`。

最终验收说明见 `docs/final-acceptance.md`，其中列出了 APK 路径、已验证项目、答辩必演示流程、云端 API 边界和已知限制。

## 运行方式

1. 用 Android Studio 打开本目录。
2. 等待 Gradle 同步完成。
3. 连接安卓手机或启动模拟器。
4. 运行 `app`。
5. 首次打开时允许相机权限。

## Android Studio 构建说明

- 推荐使用 Android Studio 自带 JDK 打开项目并同步 Gradle。
- 如果在终端运行 `./gradlew` 提示找不到 Java Runtime，需要安装 JDK 17，或在 Android Studio 中设置 Gradle JDK。
- 当前项目使用 Android Gradle Plugin 7.3.0、Kotlin 1.6.10、compileSdk 33，适合课程项目和常见 Android Studio 版本打开。
- 模型文件位于 `app/src/main/assets/facenet.tflite`，不要删除。

## 演示流程

1. 打开 App 后先看首页“下一步”提示。
2. 观察首页入场后的演示准备度、下一步提示和主要操作区是否清晰可见。
3. 在“开始演示”区域输入姓名。
4. 正对摄像头，点击“录入样本”。
5. 至少录入 2 个不同人员，每人建议录入 3 次：正脸、轻微侧脸、不同表情或光线。
6. 点击“拍照识别”，查看姓名、相似度、L2 距离和判定依据，并观察成功、未知、边界/拒识等状态切换是否清楚。
7. 点击“多人视频”，让画面中至少 2 张人脸同时入镜，等待跨帧投票确认后展示全屏相机、人脸框、扫描反馈、姓名标签和文字结果；退出时点击“停止演示”。
8. 用未录入人员测试“未知人员”结果。
9. 点击“设置与材料”，查看识别记录并生成截图摘要。
10. 在设置页点击“复制完整报告”，把摘要、技术详情、人脸库明细、完整识别记录、云端 API 演示材料和隐私与风险说明带入报告。
11. 返回主页面，关闭网络后再次识别，证明系统是本地离线运行。

## UI 与动效验收重点

- 首页入场：打开 App 后，首页应能快速呈现相机预览、演示准备度、2x2 演示流程卡、下一步提示和主要按钮；动效只用于突出流程，不应遮挡相机或文字。
- 状态切换：录入成功、识别成功、未知人员、质量不足、候选人过近和云端连接状态之间切换时，结果区应有明确视觉层级，旧结果不应造成误读。
- 全屏多人识别扫描效果：进入全屏视频模式后，相机预览应铺满主要区域，人脸框、姓名/未知标签、扫描反馈和逐张识别结果应同步表达当前识别状态。
- 小屏不重叠：在小屏手机上，按钮、状态文案、相机预览、人脸标签和底部结果区不应互相压住；长姓名或较长提示文案需要换行或截断后仍可理解。
- 视觉截图：答辩报告建议保留首页入场、状态切换、全屏多人识别和小屏布局截图；真机结论必须来自实际设备截图或录屏，不能用预期效果代替。

## 答辩前最后检查

1. 确认 `app/build/outputs/apk/debug/app-debug.apk` 已生成。
2. 用真机完整走一遍录入、识别、未知人员、多人视频和断网识别，并保留截图或录屏。
3. 每位测试人员至少录入 3 次，现场光线保持稳定，并记录同人最大 L2、跨人最近 L2 和阈值建议。
4. 检查首页入场、状态切换、全屏多人识别扫描效果和小屏不重叠；如果没有小屏真机，需在验收记录中标为待测。
5. 进入“设置与材料”生成一次截图摘要，并确认验收覆盖至少包含识别成功、未知人员和视频多人识别。
6. 若演示云端对比，在“设置与材料”提前测试 Face++ 或 CompreFace 连接，并使用独立 FaceSet outer_id；若连接失败，直接回到本机演示模式，不现场排查 Key、额度或服务端问题。
7. 准备好 `docs/final-acceptance.md`、`docs/report-outline.md` 和 `docs/technical-design.md` 作为报告素材。

## 云端 API 模式

项目新增“云端 API”识别模式。当前代码支持两种云端提供商：Face++ 托管云端 API 和自部署 `exadel-inc/CompreFace` REST 服务。如果不想本地部署，优先选择 Face++，通常可以用免费 Key 做课程演示；CompreFace 只是客户端已接好，服务端没有随 APK 一起部署。

云端模式提供“测试云端连接”按钮。Face++ 会初始化/确认 FaceSet 是否可用；CompreFace 会读取人脸库 subject 列表，用于提前确认服务地址和 API Key 是否可用。

连接测试后，云端设置区会保留最近一次测试状态，包括测试时间、当前/上次测试提供商和成功或失败原因。切换 Face++ 与 CompreFace 后，如果上次状态来自另一个提供商，界面会提示重新测试，避免把旧结果误当成当前服务可用。

云端 API Key/API Secret 只用于课程演示时临时填写。当前 App 不硬编码密钥，但会把用户填写的云端配置保存在本机偏好设置中，正式产品应改为服务端代理或加密密钥管理。

Face++ 默认服务地址：

```text
https://api-us.faceplusplus.com
```

Face++ 云端模式需要填写：

- API Key
- API Secret
- FaceSet outer_id，例如 `final_project_faces`

模拟器访问电脑本机 CompreFace 服务时，服务地址通常填写：

```text
http://10.0.2.2:8000
```

真机访问同一局域网电脑服务时，填写电脑局域网 IP，例如：

```text
http://192.168.1.10:8000
```

详细部署和报告写法见 `docs/cloud-api-integration.md`。

注意：当前项目没有内置或自动部署 CompreFace 服务。CompreFace 是服务端组件，需要单独 Docker 部署；如果希望最省事，建议优先使用 Face++ 云端 API，而不是临时搭服务端。

## 技术说明

人脸识别流程：

1. CameraX 获取拍照图像。
2. ML Kit Face Detection 检测人脸框。
3. 拍照录入/拍照识别要求画面中只有一张人脸；视频多人识别允许同一画面中存在多张人脸。
4. 根据人脸框裁剪出人脸图像。
5. FaceNet TFLite 模型输出 128 维特征向量。
6. 使用 L2 距离比较当前向量和本地人脸库。
7. 距离低于阈值时识别为对应人员，否则识别为未知人员。
8. 如果第一候选人和第二候选人的距离过近，即使低于阈值也拒绝确认，避免把相似人员误判为同一人。

默认 L2 阈值为 `10.0`，来自参考项目中的经验配置。当前版本不只看单一最小距离，还会综合最近样本、平均距离和候选人间隔，输出安全区、稳定区、边界区或拒识区说明。实际演示时可以通过多录入同一个人的不同角度照片来提升稳定性。

## 3 分钟现场演示脚本

1. 说明项目是本地离线识别：CameraX + ML Kit + FaceNet TFLite，不调用云 API。
2. 展示首页入场后的人脸库状态、L2 阈值和“下一步”提示，说明 UI 会把答辩流程聚焦在主操作区。
3. 输入姓名，点击“录入样本”录入 A；再录入 B，每人补录 2-3 次。
4. 对 A 点击“拍照识别”，解释姓名、相似度、L2 距离和安全区/边界区说明，同时展示结果状态切换。
5. 点击“多人视频”，展示全屏预览、扫描反馈、多张脸的人脸框和逐张输出结果；退出时点击“停止演示”。
6. 用未录入人员或相似候选场景展示“未知人员/拒绝确认”。
7. 关闭网络后再次识别，说明数据和模型都在本机。
8. 点击“设置与材料”，展示识别记录和测试摘要，说明可直接用于报告截图。
9. 点击“复制完整报告”，说明剪贴板会包含人脸库明细、完整识别记录、云端 API 演示材料、隐私与风险说明和报告使用建议。
10. 可选在“设置与材料”切换到“云端 API”模式，优先选择 Face++ 并填写 API Key、API Secret 和 FaceSet outer_id；也可选择 CompreFace 自部署服务，展示同一套拍照流程能调用线上服务识别。

## 常见问题

- 没有检测到人脸：保持正对摄像头，保证光线充足。
- 提示画面偏暗、过亮或模糊：调整光线，保持手机稳定后重新录入或识别。
- 检测到多张人脸：画面中只保留一个人。
- 识别成未知人员：同一个人多录入 2-3 次，覆盖不同角度和表情。
- 提示人脸库需补强：按演示准备度提示补录样本或增加对照人员。
- 提示样本差异偏大：说明同一人的多次录入不够一致，建议删除该人员后在稳定光线下重新录入。
- 提示人员区分度偏低：说明两位人员的人脸特征太接近，建议分别补录更清晰的正脸样本。
- 阈值校准建议提示样本不足：先完成至少 2 位人员、每人 2 组样本，再生成测试摘要。
- 模型加载失败：确认 `app/src/main/assets/facenet.tflite` 存在。
- 终端无法构建：优先使用 Android Studio 自带 JDK；终端环境需要单独安装 JDK。
