# 演示清单

## 演示前准备

- 准备 2-3 位测试人员。
- 每人录入 2-3 次，尽量覆盖正脸、轻微侧脸、不同表情。
- 保证现场光线充足，优先选择正面柔和光；避免逆光、顶光过强、屏幕反光和背景中出现无关人脸。
- 录入时避免逆光、过暗、手抖和人脸贴边；如果 App 提示质量不足，按提示调整后重拍。
- 提前确认 `app/src/main/assets/facenet.tflite` 存在。
- 先用 Android Studio 安装到手机并完整走一遍流程；如果命令行 `adb devices` 没有列出设备，也必须在 Android Studio 的设备列表中确认真机可运行。
- 现场主线默认使用“本机演示（推荐）”；云端 API 只作为加分对比，网络或 API Key 不稳定时直接跳过。
- 演示断网识别前先完成本地录入和至少一次成功识别，再打开飞行模式或关闭 Wi-Fi/移动数据。
- 准备一台常规尺寸手机和一台小屏或窄屏设备；如果现场只有一台设备，小屏不重叠只能标为待真机补测，不能写成已通过。
- 提前录制或截图首页入场、状态切换、全屏多人识别扫描效果和小屏布局，作为 UI/动效验收材料。

## 必备截图

- 首页和相机预览。
- 首页入场后的首屏状态：相机预览、演示准备度、下一步提示和主要按钮都应可见。
- “设置与材料”页，证明云端配置、识别记录、报告素材和数据管理没有挤在主演示页面。
- 输入姓名并录入成功。
- 已录入人数、每人特征组数和当前阈值。
- 首页演示准备度，包含人脸库健康状态和补录建议。
- 首页“下一步”提示，证明 App 会引导演示者按录入、识别、未知人员、多人视频和摘要顺序完成验收。
- 测试摘要中的样本一致性风险，证明系统会检查录入样本是否稳定。
- 测试摘要中的人员区分度风险，证明系统会检查不同人员是否容易混淆。
- 测试摘要中的阈值校准建议，用于说明默认阈值和当前人脸库质量的关系。
- 测试摘要或完整报告中的算法样本评测说明，用于记录同人 L2、跨人 L2、未知人员 L2 和风险控制信号。
- 完整报告中的真机证据链清单，用于区分已有记录、待真机补证据和云端可选项目。
- 测试摘要中的验收覆盖，检查识别成功、未知人员、边界/拒识、候选人过近和视频多人识别是否已经准备材料。
- “设置与材料”页中的云端 API 连接状态，截图展示最近测试时间、提供商和成功/失败原因；如果不演示云端，可截图说明云端是可选第二模式。
- 识别成功结果。
- 未知人员结果。
- 状态切换过程，至少覆盖录入成功到识别成功、识别成功到未知人员或拒识提示。
- 全屏视频多人识别结果，包括铺满屏幕的相机预览、人脸框和姓名/未知标签。
- 全屏多人识别扫描效果，包括扫描反馈、人脸框、姓名/未知标签和底部逐张结果不互相遮挡。
- 安全区、边界区、拒识区或候选人过近的判定说明。
- 人脸质量不足提示，例如偏暗、过亮、模糊或贴边。
- 未检测到人脸提示。
- 拍照识别时的多张人脸提示；视频多人识别本身允许同画面多张人脸。
- 识别记录。
- 测试摘要，包括录入人数、特征组数、识别成功次数、未知人员次数、平均相似度、边界匹配和拒识统计。
- 关闭网络后仍可识别。
- 小屏手机或窄屏模式截图，确认首页、设置页和全屏多人识别中的按钮、长文案、人脸标签和结果区不重叠。

## 答辩演示顺序

1. 打开 App，说明这是本地离线识别。
2. 展示首页入场后的已录入人数、L2 阈值和“下一步”提示，说明首屏聚焦录入、识别和多人视频三个核心动作。
3. 输入姓名并点击“录入样本”录入第一个人。
4. 按提示录入第二个人，并给每人补录 2-3 次。
5. 对第一个人点击“拍照识别”，展示姓名、相似度、L2 距离和识别结果状态切换。
6. 点击“多人视频”进入全屏演示，让至少两张人脸同时入镜，展示系统能准实时分析铺满屏幕的相机画面、绘制人脸框、显示扫描反馈并逐张识别多个人脸；说明姓名标签会经过跨帧投票和平滑显示，减少单帧误判，退出时点击“停止演示”。
7. 对未录入人员拍照，展示未知人员。
8. 故意让画面中没有人脸，展示异常提示。
9. 打开飞行模式或关闭网络，再次识别，证明不依赖云 API。
10. 点击“设置与材料”，展示识别记录。
11. 在设置页点击“生成截图摘要”，截图放入报告测试结果部分。
12. 点击“复制完整报告”，把统计文字、人脸库明细、完整识别记录、云端 API 演示材料和隐私与风险说明粘贴到报告或答辩稿。
13. 可选在设置页切换到“云端对比”，选择 Face++ 或 CompreFace，点击“测试云端连接”，展示最近连接状态；如果现场网络不可用，说明本地离线模式仍是主交付。

## 真机检查项

- 设备信息：记录品牌、型号、Android 版本、屏幕尺寸或分辨率。
- 安装启动：使用 Android Studio 或可用 `adb` 安装 `人脸识别演示台-debug.apk` 或 `app/build/outputs/apk/debug/app-debug.apk`，记录是否首次启动成功和是否正常申请相机权限。
- 主流程：录入 A/B、识别 A/B、未知人员、无人脸提示、拍照多人脸提示、全屏多人视频、断网识别都需要真实截图或录屏。
- UI/动效：记录首页入场、状态切换、全屏多人扫描效果和小屏不重叠；没有对应设备时只能标为待测。
- 性能体感：记录全屏多人视频是否明显卡顿、是否出现黑屏、相机预览拉伸或人脸框明显错位。
- 云端对比：如演示 Face++ 或 CompreFace，记录测试时间、提供商、成功/失败原因；不演示云端时标为未启用，不写成功。

## 风险兜底

- 如果现场识别不稳定，先让同一个人多录入 2-3 次。
- 答辩正式录入建议每人准备 3 组：正脸、轻微侧脸、不同表情或光线，并记录同人最大 L2、跨人最近 L2 和阈值建议。
- 如果多人视频识别标签还在“识别中”，保持两张脸稳定入镜 3-5 秒，等待跨帧投票确认后再截图。
- 如果全屏多人识别扫描反馈遮挡人脸或标签，优先保留稳定的人脸框和姓名标签截图，把动效问题记录为待优化。
- 如果小屏出现按钮或文字拥挤，不现场改代码；记录设备型号、截图和遮挡位置，答辩时说明代码已有紧凑保护，但该设备仍列入后续兼容优化。
- 如果断网后云端模式报错，切回“本机演示（推荐）”再识别，说明最低交付不依赖网络。
- 如果云端连接失败，不现场排查 Key、额度或服务端，直接展示中文错误提示、最近一次连接状态和本地离线主线；Face++ 建议答辩前使用独立 FaceSet outer_id，避免旧数据混入。
- 如果模拟器摄像头效果差，优先使用真机。
- 如果 Android Studio 同步失败，检查 Gradle JDK 是否可用。
- 如果老师问为什么没有活体检测，说明本项目优先完成身份识别，活体检测作为后续安全增强。
