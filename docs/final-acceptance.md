# 最终验收说明

## 交付结论

当前版本可以作为人脸识别期末作业的主体交付版本。项目已经完成本机离线人脸识别闭环，并补充了云端 API 对比模式、演示引导、测试摘要、报告素材和风险说明。本机模式不上传人脸图片或特征；云端模式只有用户主动切换并填写配置后才会上传裁剪后的人脸图。

最低交付主线是本机离线模式：Android 端使用 CameraX 拍照和预览，ML Kit 检测人脸，TensorFlow Lite FaceNet 模型提取 128 维特征向量，再用本地 JSON 人脸库和 L2 距离完成人员匹配。云端 API 模式只是第二演示模式，不影响本机模式交付。

Debug APK 交付物为 `app/build/outputs/apk/debug/app-debug.apk`。项目根目录另有便于提交和传输的副本 `人脸识别演示台-debug.apk`。该 APK 适合课程验收、真机安装和现场演示，不等同于签名发布版；如果课程平台要求 release APK 或特定签名包，需要按平台要求另行打包。

当前根目录 APK 的 SHA-256 为 `f1098334dae660288223ea7b13c420d2ccc60f952abcdcf9a5fc34e5a21813eb`，用于提交后核对文件是否被替换。

## 当前版本能力

- 拍照录入人员姓名和人脸特征。
- 拍照识别已录入人员，显示姓名、相似度、L2 距离和判定说明。
- 未录入人员显示未知人员。
- 录入和单人拍照识别会拦截无人脸、多张人脸、低质量人脸、模型未加载和相机异常。
- 多人视频演示支持预览画面中的多张人脸框和姓名/未知标签。
- 多人视频演示记录会附带紧凑调试摘要，包含轨迹号、匹配方式、投票数、确认阈值和 L2 距离，便于解释跨帧稳定。
- 首页显示人脸库状态、阈值、演示准备度和下一步提示。
- UI/动效验收重点已补入文档：首页入场、状态切换、全屏多人识别扫描效果和小屏不重叠。
- 识别记录、本机人脸库、测试摘要、算法样本评测、真机证据链和完整报告素材均可在 App 内生成。
- 云端 API 对比模式支持 Face++ 托管云端 API 和 CompreFace-compatible REST 服务。
- 云端连接测试会记录最近测试时间、提供商和成功/失败原因。

## UI/动效验收范围

本次文档把 UI/动效验收拆成四个高优先级项目，作为答辩前真实设备检查标准：

1. 首页入场：冷启动后首页应快速进入可操作状态，相机预览、演示准备度、2x2 演示流程卡、下一步提示和主要按钮清晰可见。
2. 状态切换：录入成功、识别成功、未知人员、质量不足、候选人过近和云端连接状态之间切换时，结果区应保持可读，旧状态不应误导当前判断。
3. 全屏多人识别扫描效果：至少两张人脸同时入镜时，全屏预览、人脸框、扫描反馈、姓名/未知标签和逐张识别结果应同步可见。
4. 小屏不重叠：小屏或窄屏设备上，按钮、长文案、人脸标签、相机预览和底部结果区不应互相遮挡。

当前可确认的是：验收标准和记录表已写入 `README.md` 与 `docs/demo-checklist.md`。当前不可确认的是：这些效果在具体真机型号上的实际表现；需要用真实设备截图或录屏补齐，不得用预期描述替代测试结果。

测试命令边界如下：`testDebugUnitTest` 是本地 JVM 单元测试；`assembleDebug` 是 Debug APK 构建；`assembleDebugAndroidTest` 只是 Android 测试 APK 构建；`connectedDebugAndroidTest` 才是设备侧执行。没有设备运行记录时，只能写“待验证”或“未执行”；不能用本地单元测试、APK 构建成功、静态检查或预期流程替代真实设备结果。

## 答辩必演示流程

1. 打开 App，说明默认使用“本机演示（推荐）”，不依赖云账号和网络。
2. 展示首页入场后的相机预览、人脸库状态、演示准备度和下一步提示。
3. 输入第一位测试人员姓名，点击“录入样本”录入当前人员。
4. 录入第二位测试人员，每人建议补录 2-3 次。
5. 对已录入人员拍照识别，截图姓名、相似度、L2 距离、判定说明和状态切换。
6. 用未录入人员拍照识别，截图未知人员结果。
7. 开启多人视频演示，让至少两张人脸同时入镜，截图全屏预览、人脸框、扫描反馈和姓名/未知标签。
8. 故意让画面无人脸或多人脸，展示中文错误提示。
9. 关闭网络后再次使用本机识别，证明最低交付不依赖云端。
10. 生成截图摘要，展示验收覆盖和下一步建议。
11. 复制完整报告素材，粘贴到课程报告或答辩稿。

## 云端 API 是否必需

云端 API 不是最低交付要求。它的定位是加分或对比模块，用于说明系统可以扩展到线上服务化架构。

- 推荐云端方案：Face++ 托管云端 API，适合不想自己部署服务端的课程演示。
- 备选云端方案：CompreFace-compatible REST 服务，适合说明开源自部署架构。
- 当前项目没有内置或自动部署 CompreFace 服务；如果选择 CompreFace，需要单独准备 Docker 服务端。
- 云端模式可能受网络、免费额度、API Key、FaceSet 配置影响；现场答辩应优先保证本机离线模式可完整演示。
- 云端错误提示已按常见原因中文化；如果出现 Key、额度、网络或服务端问题，现场直接切回本机离线模式。

## 已知限制与答辩解释

- 当前不是商业级门禁或金融核身系统，准确率会受光线、角度、摄像头质量和样本数量影响。
- 首页入场、状态切换、全屏多人识别扫描效果和小屏不重叠目前是文档化验收标准，尚未在当前终端环境获得真机证据。
- 真机证据链清单只是报告辅助材料，不能替代真实设备截图、录屏、设备型号和执行记录。
- 没有把活体检测纳入最低交付，因此不能防止照片翻拍攻击；报告中将活体检测作为后续安全增强。
- 本机人脸库使用 SharedPreferences + JSON，适合作业演示；正式产品可升级为 Room 或后端数据库。
- 当前 APK 已关闭 Android 自动备份，降低演示数据被系统备份带走的风险；正式产品仍应增加加密存储、数据导出控制和清理策略。
- 云端 API Key 只适合课程演示时临时填写；当前配置保存在本机偏好设置中，正式系统不应把密钥保存在客户端。
- GPT 多模态模型适合图片理解和说明，不适合作为“这个人是谁”的人脸库身份比对引擎。
